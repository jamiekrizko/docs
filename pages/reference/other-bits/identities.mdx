## Identities

### Create Identity
```sh
curl --request POST \
     --url 'https://api.mixpanel.com/track#create-identity' \
     --header 'accept: text/plain' \
     --header 'content-type: application/x-www-form-urlencoded' \
     --data 'data={
      "event": "$identify",
      "properties": {
          "$identified_id": "ORIGINAL_ID",
          "$anon_id": "NEW_ID",
          "token": "YOUR_PROJECT_TOKEN"
      }
}
'
```

To see this API in action, visit our [API Playground](https://developer.mixpanel.com/reference/create-identity).
<CustomCallout type="info">
    You can also use the import endpoint: https://api.mixpanel.com/import/
</CustomCallout>

### Create Alias
```sh
curl --request POST \
     --url 'https://api.mixpanel.com/track#identity-create-alias' \
     --header 'accept: text/plain' \
     --header 'content-type: application/x-www-form-urlencoded' \
     --data 'data={
    "event": "$create_alias",
    "properties": {
        "distinct_id": "other_distinct_id",
        "alias": "your_id",
        "token": "YOUR_PROJECT_TOKEN"
    }
}
'
```

To see this API in action, visit our [API Playground](https://developer.mixpanel.com/reference/identity-create-alias).

<CustomCallout type="info">
    You can also use the import endpoint: https://api.mixpanel.com/import/
</CustomCallout>

Mixpanel supports adding an alias to a distinct id. An alias is a new\nvalue that will be interpreted by Mixpanel as an existing value. That\nmeans that you can send messages to Mixpanel using the new value, and\nMixpanel will continue to use the old value for calculating funnels and\nretention reports, or applying updates to user profiles.

### Merge Identities
```sh
curl --request POST \
     --url 'https://api.mixpanel.com/import?strict=1' \
     --header 'accept: application/json' \
     --header 'content-type: application/json'
```

To see this API in action, visit our [API Playground](https://developer.mixpanel.com/reference/identity-merge).

<CustomCallout type="error" title="Merging identities is irreversible">
    `$merge` is a very powerful tool, so we will only accept `$merge` events that are sent via `https://api.mixpanel.com/import`, which is protected by the project api secret. You **cannot** unmerge `distinct_id`.
</CustomCallout>
